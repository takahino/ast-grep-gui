ast-grep GUI — パターン記法（詳細）

このドキュメントはアプリに埋め込まれています。公式ドキュメントと併せて参照してください。

1. AST パターンとは

「AST 検索 / AST そのまま」モードでは、ソースコードを構文木（AST）として解析し、パターン文字列で木の形を指定してマッチ箇所を探します。テキストの部分一致（プレーンテキスト）や正規表現モードとは別の仕組みです。

  • パターンは ast-grep が解釈するコードの断片として書きます（コメントや空白の扱いは言語・ノード依存）。
  • メタ変数（$NAME など）で「ここに任意の式や文が入る」といった穴を開けられます。

2. メタ変数一覧

書き方意味
$VAR単一のノードにマッチ（式・識別子・型など、位置による）
$$$NAME0 個以上の連続するノードにマッチ（引数リスト、複数文など）
$_マッチはするがキャプチャしない「ワイルドカード」
同じ名前の $A出現箇所同士が同一のコードに対応する必要がある（同一性制約）

変数名は英字で始まる識別子として扱われます。詳細な構文は公式の Pattern Syntax を参照してください。

3. 記述のコツ

  • 言語の選択：ツールバーで Rust / Python などを選ぶと、その言語のパーサで AST が構築されます。Auto は拡張子から推定します。違う言語でパースすると期待したノードにならずマッチしません。
  • マッチ範囲：この GUI の AST モードでは、ast-grep CLI と同様にノードのバイト範囲が使われます。
  • $$$ の使い所：可変長の引数、match のアーム列、ブロック内の複数文など、文法上「複数子ノード」が並ぶ場所向けです。単一式だけを拾いたいときは $EXPR のように単一メタ変数にします。
  • うまくいかないとき：まずパターンを小さくし、一段階ずつ広げて試してください（例：まず fn $NAME だけ、次に引数まで）。
プレーン / 正規表現モードでは AST は使いません。文字列検索・正規表現のルールが適用されます。

4. 言語別プリセット（GUI と対応）

ヘルプウィンドウの「プリセット」は言語モードに応じて切り替わります。参考までに代表的なものを列挙します。

Rust

  • fn $NAME($$$ARGS) — 関数定義
  • impl $TRAIT for $TYPE — トレイト実装
  • $VAR.unwrap() — unwrap 呼び出し
  • $VAR.clone() — clone 呼び出し
  • panic!($$$ARGS) — panic マクロ

Java

  • $VAR == null
  • System.out.println($$$ARGS)

Python

  • print($$$ARGS)
  • import $MODULE

JavaScript / TypeScript

  • console.log($$$ARGS)

Go

  • if err != nil { $$$BODY }

Auto / その他

$EXPR のように汎用の単一ノードで広く試すこともできます。

5. 追加の記述例

環境や ast-grep のバージョンにより、細部の解釈は変わる場合があります。次は出発点として試せます。

Rust

struct $NAME { $$$FIELDS }
impl $TYPE { $$$ITEMS }
Some($INNER)
if let $PAT = $EXPR { $$$THEN }

Python

def $NAME($$$ARGS):
    $$$BODY
class $NAME($$$BASES):
    $$$BODY

TypeScript / JavaScript

function $NAME($$$ARGS) { $$$BODY }
const $NAME = ($$$ARGS) => $BODY
export default $EXPR

6. トラブルシューティング

  • ヒットゼロ：パターンがその言語の構文として成立しているか、言語・拡張子が合っているかを確認してください。
  • ファイルフィルタ：空なら（AST モードでは）言語既定の拡張子のみが対象です。Plain/Regex では別ルールです。
  • コンテキスト行数：表示・エクスポートの前後行の広さだけを変えます。マッチの有無自体は変えません。
