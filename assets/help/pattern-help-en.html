ast-grep GUI — Pattern syntax (full help)

This page is bundled with the app. Use it together with the official ast-grep documentation.

1. What is an AST pattern?

In AST search and AST raw modes, source code is parsed into a syntax tree and your pattern string describes a shape in that tree. This is different from plain-text search or regex modes.

  • Patterns are written as code-like fragments interpreted by ast-grep (comments/whitespace handling depends on language and node).
  • Meta-variables such as $NAME stand for “any expression / identifier / …” at that position.

2. Meta-variables

SyntaxMeaning
$VARMatches a single node (expression, identifier, etc., depending on position)
$$$NAMEMatches zero or more consecutive nodes (argument lists, multiple statements, …)
$_Matches without capturing (wildcard)
Same name $AAll occurrences must match the same code (equality constraint)

See the official Pattern Syntax page for full grammar details.

3. Authoring tips

  • Language selection: The toolbar language chooses the parser. Auto infers from file extensions. If the wrong parser is used, node shapes won’t match your mental model.
  • Match span: In AST modes this app aligns with ast-grep’s node byte range behavior.
  • When to use $$$: Use for variadic children—argument lists, sequences of statements, etc. Use a single meta-variable like $EXPR for one expression.
  • If matching fails: Start with a smaller pattern and grow it step by step.
Plain / Regex modes do not use the AST. Different rules apply.

  • $VAR == null

  • print($$$ARGS), import $MODULE

JavaScript / TypeScript

  • console.log($$$ARGS)

Go

  • if err != nil { $$$BODY }

Auto / others

Try a generic single-node pattern such as $EXPR.

Python

    $$$BODY

    $$$BODY

TypeScript / JavaScript

6. Troubleshooting

  • No hits: Check that the pattern is valid for that language and that language/extension filters line up.
  • File filter: When empty, AST mode uses language-default extensions (see in-app mapping). Plain/Regex modes differ.
  • Context lines: Only changes preview/export context around matches, not whether a match exists.
